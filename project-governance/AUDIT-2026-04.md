### A4. `wisehire-validate-early-access` missing from §9

- **Source proof:** `supabase/functions/wisehire-validate-early-access/` exists (14th function).
- **Fix applied:** Added it to §9's WiseHire Edge Functions table; updated heading to "(14 functions)."

### A5. WiseHire Phase 2/3 status was overstated; public job board was never shipped
- **Claim in governance (pre-audit):** ARCHITECTURE.md §9 listed routes `/jobs`, `/jobs/:companySlug`, `/jobs/:companySlug/:roleSlug`, `/my-applications`, plus 17 WiseHire tables including `wisehire_bulk_screen_jobs`, `wisehire_scorecards`, `wisehire_scorecard_templates`, `wisehire_candidate_notes`, `wisehire_outreach_emails`, `wisehire_clients`, `wisehire_saved_searches`, `wisehire_applications`, `talent_pool_profiles`, `talent_pool_views`. PRODUCT.md described Phase 2 and Phase 3 as "shipped" / "partially shipped."
- **Source proof:**
  - Route registration: `grep -n "Route path=" src/AppInterior.tsx` enumerates every WiseHire path; **no `/jobs` or `/my-applications` route is registered.** The only `/jobs` reference in the codebase is `src/components/layout/RedirectJobRoute.tsx`, a helper that maps `/jobs/:id → /job/:id` (the legacy private WiseResume job-application detail route).
  - Migrations: `ls supabase/migrations/ | grep -E "wisehire|talent_pool"` returns only the 8 Phase 1 files (`20260420000001` … `20260420000008`) plus the supporting RPC/error_log/audit-log migrations and `20260422000001_pipeline_bulk_and_master_cv_rpcs.sql` / `20260422000002_atomic_refund_credit_and_reset_premium_usage.sql`. **No migration creates** `wisehire_bulk_screen_jobs`, `wisehire_scorecards`, `wisehire_scorecard_templates`, `wisehire_candidate_notes`, `wisehire_outreach_emails`, `wisehire_clients`, `wisehire_saved_searches`, `wisehire_applications`, `talent_pool_profiles`, or `talent_pool_views`.
  - Edge functions DO exist (`wisehire-bulk-screen`, `wisehire-mask-cvs`, `wisehire-send-outreach`, `wisehire-talent-search`, `wisehire-talent-view`, `wisehire-apply`) and call those tables via `.from('…')` — meaning these code paths will fail at runtime with "relation does not exist" until the migrations land.
- **Fix applied:**
  - **PRODUCT.md §3** rewritten: Phase 1 marked "shipped, end-to-end working" with each bullet citing both edge function and table. Phase 2 and Phase 3 marked "partially shipped (edge + frontend code present; DB migrations missing)" with each bullet flagging the missing tables. The public job board bullet explicitly says "**NOT shipped**" and references the `RedirectJobRoute` helper.
  - **ARCHITECTURE.md §9 routes table:** removed `/jobs`, `/jobs/:companySlug`, `/jobs/:companySlug/:roleSlug`, and `/my-applications`. Added an inline note explaining the removal and pointing back to this audit item.
  - **ARCHITECTURE.md §9 WiseHire Tables block:** updated heading from "(17 tables, all with RLS)" to a split count plus a clear note that 9 of the listed tables have **no migration file**. The table rows themselves remain in the doc as a reference for the planned schema, but the warning is unmissable.
  - **DECISIONS.md ADR #11 (proposed in this audit's first pass) removed** because the underlying decision — "ship a first-party public job board" — is not implemented in the codebase. There is nothing live to record an ADR against.
  - **CONSTITUTION.md §5** decision count updated from 8 → 10 (reflecting only ADRs #9 and #10, which ARE backed by shipped code: `creditLimits.json` and `atomic_credit_deduction.sql`).
  - **CHANGELOG 2026-04-20 entry** rewritten to remove "Phase 3 follow-on (also live)" claims and instead state Phase 1 is fully shipped, Phase 2/3 are code-only pending migrations.
