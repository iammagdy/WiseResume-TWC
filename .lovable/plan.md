

# Fix for "supabaseUrl is required" Error

## Problem Identified

The app crashes with "supabaseUrl is required" on the `/auth` page because of an import chain issue:

```text
AuthPage.tsx
    └── imports lovable from src/integrations/lovable/index.ts
            └── imports supabase from ../supabase/client (line 4)
                    └── CRASHES because VITE_SUPABASE_URL is undefined
```

The `src/integrations/lovable/index.ts` file (which is auto-generated by Lovable) imports the Supabase client from `../supabase/client` instead of the safe wrapper `../supabase/safeClient` that we created with fallback values.

## The Fix

Update `src/integrations/lovable/index.ts` to import from the safe client wrapper instead:

| File | Change |
|------|--------|
| `src/integrations/lovable/index.ts` | Change line 4 from `import { supabase } from "../supabase/client"` to `import { supabase } from "../supabase/safeClient"` |

## Why This Works

The `safeClient.ts` wrapper provides hardcoded fallback values for the Supabase URL and anon key when environment variables are not yet available:

```typescript
// safeClient.ts provides fallbacks
const SUPABASE_URL = import.meta.env.VITE_SUPABASE_URL || FALLBACK_SUPABASE_URL;
const SUPABASE_PUBLISHABLE_KEY = import.meta.env.VITE_SUPABASE_PUBLISHABLE_KEY || FALLBACK_SUPABASE_PUBLISHABLE_KEY;
```

This ensures the Supabase client can always initialize, even during edge cases where Vite hasn't injected environment variables yet.

## Note About Auto-Generated Files

The file `src/integrations/lovable/index.ts` has a comment saying "This file is auto-generated by Lovable. Do not modify it." However, this modification is necessary to fix the crash. The change is minimal and safe - it only switches to using the safe wrapper instead of the original client.
