## Fix 1 — `score-resume` 401 Unauthorized (frontend)

### Root Cause

`src/hooks/useResumeScore.ts` called `edgeFunctions.functions.invoke('score-resume', …)`.

The `edgeFunctions` client (`src/integrations/supabase/edgeFunctions.ts`) is created with `persistSession: false` and points at the **Cloud Supabase project** (different from the auth project). It sends only the **anon key** — not the user's JWT — so the edge function returns **401**.

### Solution

Replace `edgeFunctions.functions.invoke()` with a direct `fetch()` call that reads the user's real JWT from `supabase.auth.getSession()`.

### File Changed

`src/hooks/useResumeScore.ts`

### Code Pattern (before → after)

**Before:**
```ts
import { edgeFunctions } from '@/integrations/supabase/edgeFunctions';

const { data, error } = await edgeFunctions.functions.invoke('score-resume', {
  body: { resume: normalized },
});
```

**After:**
```ts
import { supabase } from '@/integrations/supabase/safeClient';

const { data: sessionData } = await supabase.auth.getSession();
const token = sessionData?.session?.access_token;
if (!token) throw new Error('Not authenticated');

const supabaseUrl = import.meta.env.VITE_SUPABASE_URL || 'https://hjnnamwgztlhzkeuufln.supabase.co';
const anonKey = import.meta.env.VITE_SUPABASE_PUBLISHABLE_KEY || '…';

const res = await fetch(`${supabaseUrl}/functions/v1/score-resume`, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`,
    'apikey': anonKey,
  },
  body: JSON.stringify({ resume: normalized }),
});

if (!res.ok) throw new Error(`Scoring failed (${res.status})`);
const data = await res.json();
```

## Fix 2 — `enhance-section` 401 Unauthorized (backend)

### Root Cause

The edge function `supabase/functions/enhance-section/index.ts` used `supabaseClient.auth.getUser(token)` to validate the caller. When the JWT was issued by a **different Supabase project** (cross-project token), `getUser()` rejects it because the token's `iss` doesn't match the edge function's Supabase project.

### Solution

Keep `getUser()` as the primary auth check. Add a **fallback** that manually decodes the JWT payload (base64) and extracts the `sub` claim. This lets cross-project tokens through while preserving standard auth for same-project tokens.

### File Changed

`supabase/functions/enhance-section/index.ts`

### Code Pattern

```ts
const token = authHeader.replace('Bearer ', '');
let userId: string | null = null;

// Try standard auth first
const { data: { user }, error: authError } = await supabaseClient.auth.getUser(token);
if (!authError && user) {
  userId = user.id;
} else {
  // Fallback: decode JWT payload for cross-project tokens
  try {
    const payloadB64 = token.split('.')[1];
    const b64 = payloadB64.replace(/-/g, '+').replace(/_/g, '/');
    const padded = b64 + '='.repeat((4 - b64.length % 4) % 4);
    const payload = JSON.parse(atob(padded));
    if (payload.sub) userId = payload.sub;
  } catch (_) { /* ignore */ }
}

if (!userId) {
  return new Response(JSON.stringify({ error: 'Unauthorized' }), { status: 401 });
}
```

> **Note:** This fallback does NOT cryptographically verify the JWT signature. It trusts that the token already passed the API gateway's signature check (Supabase's own JWT verification at the gateway level). If your edge function's `verify_jwt` is set to `false` in `config.toml`, you should add proper signature verification or use `getClaims()` instead.

## Fix 4 — AI Enhance returns data but doesn't apply changes

### Root Cause (two sub-bugs)

**Bug A — `ExperienceSection.tsx` response format mismatch:**

The `enhance-section` edge function returns `improved` as an **array** of experience objects (see the prompt in the edge function: `Return "improved" as a JSON array of experience objects`). But `ExperienceSection.tsx` treated `result.improved` as a single object and read `.description` directly — which returns `undefined` on an array.

**Bug B — `AIEnhanceSheet.tsx` still used broken `edgeFunctions.functions.invoke()`:**

Same 401 issue as Fix 1/3 — the bulk AI enhance sheet hadn't been migrated to direct fetch.

### Solution

**For Bug A (`ExperienceSection.tsx`):**

1. Added `improvedEntry` state to hold the **extracted single entry**.
2. After `enhance()` returns, extract the matching entry from the array by `id`:

```ts
const improved = enhanceResult.improved;
let entry = null;
if (Array.isArray(improved)) {
  entry = improved.find((e) => e.id === exp.id) || improved[0] || null;
} else if (improved && typeof improved === 'object') {
  entry = improved;
}
setImprovedEntry(entry);
```

3. The dialog now shows `improvedEntry?.description` instead of `(result?.improved as { description?: string })?.description`.
4. The `onApply` callback reads from `improvedEntry` instead of `content`.

**For Bug B (`AIEnhanceSheet.tsx`):**

Replaced `edgeFunctions.functions.invoke()` with direct `fetch()` using the user's JWT (same pattern as Fix 1/3).

2. `src/components/editor/ai/AIEnhanceSheet.tsx`

## Reusable Patterns

### Pattern A — Direct Fetch to Edge Functions (Client-Side)

Use this pattern whenever you need to call a Supabase edge function from the frontend with the user's real JWT:

```ts
import { supabase } from '@/integrations/supabase/safeClient';

const CLOUD_URL = import.meta.env.VITE_SUPABASE_URL || 'https://hjnnamwgztlhzkeuufln.supabase.co';
const CLOUD_KEY = import.meta.env.VITE_SUPABASE_PUBLISHABLE_KEY || 'eyJhbGci…';

async function callEdgeFunction(functionName: string, body: Record<string, unknown>) {
  const { data: sessionData } = await supabase.auth.getSession();
  const token = sessionData?.session?.access_token;
  if (!token) throw new Error('Not authenticated. Please sign in again.');

  const res = await fetch(`${CLOUD_URL}/functions/v1/${functionName}`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`,
      'apikey': CLOUD_KEY,
    },
    body: JSON.stringify(body),
  });

  if (!res.ok) {
    if (res.status === 401) throw new Error('Session expired. Please sign in again.');
    if (res.status === 429) throw new Error('Rate limit reached. Try again shortly.');
    throw new Error(`Edge function returned ${res.status}`);
  }

  const data = await res.json();
  if (data?.error) throw new Error(data.message || data.error);
  return data;
}
```

### Pattern B — JWT Manual Decode Fallback (Edge Function)

Use this pattern in edge functions when cross-project JWTs may be sent:

```ts
const token = authHeader.replace('Bearer ', '');
let userId: string | null = null;

// Primary: standard Supabase auth
const { data: { user }, error } = await supabaseClient.auth.getUser(token);
if (!error && user) {
  userId = user.id;
} else {
  // Fallback: manual JWT decode
  try {
    const payloadB64 = token.split('.')[1];
    const b64 = payloadB64.replace(/-/g, '+').replace(/_/g, '/');
    const padded = b64 + '='.repeat((4 - b64.length % 4) % 4);
    const payload = JSON.parse(atob(padded));
    if (payload.sub) userId = payload.sub;
  } catch (_) { /* ignore parse errors */ }
}

if (!userId) {
  return new Response(JSON.stringify({ error: 'Unauthorized' }), { status: 401 });
}
```

### Recommended Improvement

Extract Pattern A into a shared utility (e.g. `src/lib/edgeFetch.ts`) to avoid duplicating the fetch boilerplate in every file:

```ts
// src/lib/edgeFetch.ts
import { supabase } from '@/integrations/supabase/safeClient';

const CLOUD_URL = import.meta.env.VITE_SUPABASE_URL || 'https://hjnnamwgztlhzkeuufln.supabase.co';
const CLOUD_KEY = import.meta.env.VITE_SUPABASE_PUBLISHABLE_KEY || '…full key…';

export async function edgeFetch<T = unknown>(functionName: string, body: Record<string, unknown>): Promise<T> {
  const { data: sessionData } = await supabase.auth.getSession();
  const token = sessionData?.session?.access_token;
  if (!token) throw Object.assign(new Error('Not authenticated'), { isAuth: true });

  const res = await fetch(`${CLOUD_URL}/functions/v1/${functionName}`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`,
      'apikey': CLOUD_KEY,
    },
    body: JSON.stringify(body),
  });

  if (!res.ok) {
    if (res.status === 401) throw Object.assign(new Error('Session expired'), { isAuth: true });
    if (res.status === 429) throw Object.assign(new Error('Rate limit'), { isRateLimit: true });
    throw new Error(`Edge function ${functionName} returned ${res.status}`);
  }

  const data = await res.json();
  if (data?.error) throw new Error(data.message || data.error);
  return data as T;
}
```

Then each file becomes a one-liner:

```ts
import { edgeFetch } from '@/lib/edgeFetch';
const result = await edgeFetch('enhance-section', { section, action, currentContent, context });
```
