# 03 – Feature Spec Template (WiseResume)

Use this to write a full spec for a new or improved feature.

```text
/speckit.specify Using the clarification for this feature, create a complete feature specification for:

<FEATURE NAME, e.g. "Voice Resume Recording in WiseResume">

Sources of truth:
- project-governance/CONSTITUTION.md
- project-governance/PRODUCT.md
- project-governance/ARCHITECTURE.md
- project-governance/WORKFLOW.md
- docs/product/PRD.md
- Legacy docs only as historical context.

The spec must include:
- Problem statement and user value.
- In-scope and out-of-scope.
- User stories with acceptance criteria.
- UX notes consistent with design rules.
- Data/API impact (high-level).
- Non-functional requirements relevant to this feature.
- Dependencies and risks.
```
