## 📁 Subdirectory Map

* **`product/`** — Product requirements, briefs, and brand guidelines (`requirements.md`, `WiseCloud-App-Brief.md`).
* **`architecture/`** — System architecture, technical context, DB schemas (`data-model.md`), and Appwrite Function specs (`appwrite-functions.md`).
* **`features/`** — Feature specifications, plans, and localization guides (dashboard, portfolio, tailoring, etc.).
* **`skills/`** — AI agent skills system, execution playbooks, and mandatory bootstrap guide.
* **`design-system/`** — Production design system (`production/`) and visual target specs (`visual-reference/`).
* **`assets/`** — Visual brand logos, architecture diagrams, UI screenshots, and mockups.
* **`qa/`** — Test suite maps (`test-suite-map.md`), machine output location specs (`test-output-locations.md`), and E2E guides.
* **`prompts/`** — AI agent system prompts, task initiation templates, and prompt engineering rules.
* **`operations/`** — Deployment runbooks for Vercel & Appwrite, key rotation, and domain routing specs.
* **`decisions/`** — Architectural Decision Records directory (`ADR` cards and decision logs).
* **`reports/`** — Categorized audit reports (`ui-ux/`, `performance/`, `devkit/`, `landing/`).
* **`security/`** — Security policies, permission protection, and credential audit specs.
* **`general/`** — General guidelines and contribution rules (`CONTRIBUTING.md`).
* **`_templates/`** — Documentation header and reference card templates.
* **`temp/`** — Temporary agent scratch directory (must be cleaned up at closeout).
* **`archive/`** — Zero-data-loss historical repository (STRICTLY NON-CANONICAL).
