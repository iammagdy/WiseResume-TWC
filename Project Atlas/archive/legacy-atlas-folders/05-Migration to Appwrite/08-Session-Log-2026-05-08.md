## Task #25 — Fix live blank page and deploy to production

**Status:** MERGED ✅

### What broke
- Live domain served old pre-Appwrite build (`index-CzJaKF4H.js`, Supabase/Kinde chunks) — blank page for all visitors.
- `public/_headers` CSP was stale (Supabase/Kinde origins, missing Appwrite).

### Root cause
`deploy-frontend.yml` used `/public_html/resume/` as the lftp mirror target. The FTP session starts at `/public_html` (the FTP virtual root = the Apache document root for `thewise.cloud`). The path `/public_html/resume/` resolved to `~/public_html/resume/` — a ghost directory outside the vhost root. Every CI deploy silently succeeded but wrote to the wrong place. The old build was sitting in the real web root `/public_html/` untouched.

### Fixes

**`.github/workflows/deploy-frontend.yml`**
- `mirror ./dist/ /public_html/resume/` → `mirror ./dist/ .` (FTP home = web root)
- Added `set net:timeout 30`, `set net:max-retries 3`, `set net:reconnect-interval-base 5`
- Added `timeout-minutes: 30` (job) and `timeout-minutes: 20` (lftp step)
- Added `rm -f index.html` before mirror (lftp skips unchanged files by size/time)
- Added **Verify build output** step (prints bundle hash from `dist/index.html`)
- Added **Probe FTP directory structure** step (prints `ls` after login)

**`public/_headers`**
- `connect-src`: removed `*.supabase.co`, `*.supabase.in`, `wss://*.supabase.co`, `*.kinde.com`; added `https://fra.cloud.appwrite.io`
- `script-src`: added `'unsafe-inline'`
- `style-src` / `font-src`: removed stale Google Fonts entries (fonts are now self-hosted)

### Outcome
- `deploy-appwrite-hubs` run `25580008577` — SUCCESS
- `deploy-frontend` run `25582827906` — SUCCESS
- `https://thewise.cloud/` now serves `index-AQhfc8ts.js` (Appwrite-native build)
