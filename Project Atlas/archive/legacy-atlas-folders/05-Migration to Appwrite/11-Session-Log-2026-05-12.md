## Work Item 1 — Real Puppeteer PDF export (server-side)

### Problem
`/api/export/pdf-native` returned 503 (`pending_appwrite_migration`). `nativePdfGenerator.ts` threw `PDFServerUnavailableError` on all three exports, falling back to `window.print()`. PDFs produced by the legacy `pdfGenerator.ts` path are image-based (html2canvas) — no selectable text.

### Root cause
The Puppeteer endpoint was explicitly stubbed as a placeholder pending an Appwrite Functions migration that hadn't been scheduled. Puppeteer `^24.42.0` was already in `package.json` but Chrome had never been installed in the container.

### Fixes

**Chrome installation**
```bash
npx puppeteer browsers install chrome
# Installed: ~/.cache/puppeteer/chrome/linux-147.0.7727.57/chrome-linux64/chrome
```

**`server/index.ts`**
- Replaced 503 stub with a full async Puppeteer implementation.
- `puppeteer.launch()` flags: `--no-sandbox`, `--disable-setuid-sandbox`, `--disable-dev-shm-usage`, `--disable-accelerated-2d-canvas`, `--no-first-run`, `--no-zygote`, `--disable-gpu`.
- Accepts `{ html: string, pageFormat: 'letter'|'a4', onePage?: boolean }` in POST body (50 MB limit pre-registered).
- `page.setViewport()` — width matches format (816 px Letter / 794 px A4) at `deviceScaleFactor: 2`.
- `page.setContent(html, { waitUntil: 'networkidle0', timeout: 30_000 })` — allows Puppeteer to load Google Fonts and CDN images before snapping.
- `page.pdf({ format, printBackground: true, margin: 0 })` — real Chrome-rendered PDF with text layer.
- `browser.close()` in `finally` — no zombie processes.
- HTTP 400 on missing `html`, HTTP 500 on Puppeteer error, HTTP 200 + `application/pdf` on success.

**`src/lib/nativePdfGenerator.ts`** (full rewrite)
- `collectDocumentStyles()` — iterates `document.styleSheets`; for same-origin sheets, inlines all CSS rules and rewrites relative `url(...)` references to absolute (`window.location.origin`-prefixed); for cross-origin sheets (Google Fonts, CDN), appends `@import url(...)` so Puppeteer fetches them directly.
- `buildSelfContainedHTML(templateHTML, css, pageFormat, opts)` — wraps template `outerHTML` in a minimal HTML doc; sets `-webkit-print-color-adjust: exact`; applies ATS-mode colour override block when `opts.atsMode` is true; sets `body { width: <pageWidthPx>px }` to match Puppeteer viewport.
- `generateNativePDF(templateEl, options)` — serialises the live DOM element, POSTs to `/api/export/pdf-native` (or `VITE_API_URL + /api/export/pdf-native` in production), returns `Blob`. Falls back to `PDFServerUnavailableError` on HTTP 503.
- `generateCoverLetterNativePDF(letter, contactInfo, options)` — delegates to existing `coverLetterPdfGenerator.ts` (already text-based pdf-lib; no server round-trip needed).
- `mergePDFBlobs(blobA, blobB)` — merges two PDFs client-side using `pdf-lib`.

**`.github/workflows/deploy-frontend.yml`**
- Added `VITE_API_URL: ${{ secrets.VITE_API_URL }}` to the build env block.
- When the Express server is deployed to a public URL, add the secret in GitHub repo settings → Secrets → `VITE_API_URL`. In dev, Vite proxy (`/api/*` → `localhost:5001`) handles it transparently.

## Work Item 2 — PreviewPage crash: `Cannot read properties of undefined (reading 'supportsPhoto')`

### Problem
`PreviewPage` crashed with `TypeError: Cannot read properties of undefined (reading 'supportsPhoto')` at line 113, cascading into `Warning: Invalid hook call` and an `ErrorBoundary` catch. The crash was reproducible on any navigation to `/preview`.

### Root cause (two related bugs)

**Bug 1 — `getTemplateConfig` has no fallback.**
```typescript
// src/lib/templateConfig.ts — before
export function getTemplateConfig(templateId: TemplateId): TemplateConfig {
  return TEMPLATE_CONFIGS[templateId]; // returns undefined for any unknown ID
}
```
Any stale, legacy, or absent template ID in Zustand persisted state returns `undefined`. The `useEffect` at line 103 reads `config.supportsPhoto` → crash.

**Bug 2 — Zustand rehydration guard skips migration for falsy values.**
```typescript
// src/store/resumeStore.ts — before
if (state && state.selectedTemplate) {          // skipped when selectedTemplate is undefined / ''
  state.selectedTemplate = migrateTemplateId(state.selectedTemplate);
}
```
If a user's localStorage had `selectedTemplate: undefined` or `""` (old format), `migrateTemplateId` was never called and the unmigrated value was passed directly to `getTemplateConfig`.

### Fixes

**`src/lib/templateConfig.ts`**
```typescript
export function getTemplateConfig(templateId: TemplateId): TemplateConfig {
  return TEMPLATE_CONFIGS[templateId] ?? TEMPLATE_CONFIGS['modern'];
}
```

**`src/store/resumeStore.ts`**
```typescript
if (state) {
  // Always migrate — removes falsy guard so undefined / '' is also normalised
  state.selectedTemplate = migrateTemplateId(state.selectedTemplate);
}
```

### Verification
- HMR applied cleanly, no further errors in console.
- `npx tsc --noEmit` — clean.
