# Project Atlas — Historical Archive

> [!CAUTION]
> **HISTORICAL ONLY — NOT SOURCE OF TRUTH**
>
> * This directory and all its subdirectories contain **historical, legacy, or superseded documentation and audit reports only**.
> * **It is NOT a source of truth.**
> * **AI agents and developers MUST NOT use any file in this archive as current system truth** unless explicitly instructed by the project owner.
> * Current system truth lives **exclusively** in the living specs under `Project Atlas/` (`MASTER_HANDBOOK.md`, `CURRENT_STATE.md`, `RULES.md`, `architecture/`, `features/`, `product/`, `deployment/`, etc.).
> * Files in this archive are preserved solely for safety, auditing, and zero-data-loss traceability.

---

## Archive Structure

* `legacy-docs/` — Legacy root/repository documentation files (e.g. historical `CHANGELOG.md`, duplicate briefs).
* `legacy-atlas-folders/` — Legacy pre-Appwrite migration folders and pre-normalization specification sets (e.g. `00-Full-App-Reference`, `01-Currently Implemented`).
* `historical-audits/` — Historical point-in-time system, UI/UX, QA, security, and performance audit folders.
* `imported-repo-docs/` — Archived documentation files imported from scattered repo locations.
* `imported-reports/` — Archived report files imported from scattered repo locations.
* `review-needed/` — Files undergoing review or classification.
