# Package Manifest

This is the complete WiseResume / WiseHire design-system pack.

## Main files

| File / Folder | Purpose |
|---|---|
| `design.md` | Single-file brief for AI agents and design tools. Start here. |
| `README.md` | Full design-system overview and product context. |
| `VISUAL_REFERENCE_GUIDE.md` | Explains how the interactive preview connects to production docs. |
| `DESIGN_SYSTEM.md` | Main production design-system rules. |
| `DESIGN_TOKENS.md` | Token definitions and implementation mapping. |
| `FONT_SYSTEM.md` | Full font rules, loading strategy, Tailwind mapping, and typography scale. |
| `COMPONENT_LIBRARY.md` | Core and product-specific component specs. |
| `PRODUCT_FLOWS.md` | Key WiseResume and WiseHire flows. |
| `MOBILE_RULES.md` | Mobile-first behavior and QA rules. |
| `ACCESSIBILITY.md` | Accessibility requirements. |
| `IMPLEMENTATION_GUIDE.md` | Safe phased rollout plan. |
| `AUDIT_CHECKLIST.md` | Audit checklist before coding. |
| `CLAUDE_CODE_PROMPTS.md` | Ready prompts for AI coding agents. |
| `interactive-preview/` | Interactive visual preview from the new WiseResume.zip. |
| `preview/` | Static preview cards from the original design-system package. |
| `ui_kits/` | WiseResume and WiseHire UI kit references. |
| `assets/` | Logos, favicons, marks, and brand assets. |
| `fonts/` | Font implementation references only; no binary font files. |
| `colors_and_type.css` | Portable CSS token reference. |

## Best use

- For design AI: read `design.md`, then open the interactive preview.
- For coding AI: read `design.md`, then audit using `AUDIT_CHECKLIST.md`, then plan with `IMPLEMENTATION_GUIDE.md`.
- For humans: start with `README.md`, then use `VISUAL_REFERENCE_GUIDE.md`.
