# Repo-level documentation surfaces

**Last verified:** 2026-05-08

These are top-level directories at the repo root that hold reference material, governance, scripts, and tests — not runtime code. Atlas indexes them here so a new contributor knows what each folder means and which is canonical for which concern.

| Card | Folder | Purpose |
|---|---|---|
| `project-governance.md` | `project-governance/` | Constitution, ADRs, branding, workflow — canonical platform rules. |
| `routing-ai-providers.md` | `Routing AI Providers/` | Planning docs for the AI routing rebuild. |
| `specs.md` | `specs/` | SpecKit feature specs (WiseHire HR + Wise AI Agent Evolution). |
| `docs.md` | `docs/` | Engineering docs: backend architecture, AI tools audit, ops runbooks, screenshots. |
| `reports.md` | `reports/` | Audit / e2e / drift reports. |
| `scripts.md` | `scripts/` | CI/CD + maintenance scripts. |
| `tests.md` | `tests/` | Repo-level e2e + model-comparison harnesses. |
| `wise-templates.md` | `wise-templates/` | Markdown templates for issues / changelogs / specs. **Not** resume templates. |

> Resume templates live under `src/components/templates/`, not in `wise-templates/`.
