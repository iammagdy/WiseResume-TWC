## Task #21 — Datadog eval metrics for AI answer quality (MERGED, task agent)

**Status:** MERGED ✅

### What was done

**`appwrite-hubs/ai-gateway/src/main.js`**
- Added `llmobs.submitEvalMetric()` calls inside `llmobs.trace()` callback, after a successful AI response.
- Two metrics emitted per successful span:
  - `response_word_count` (score) — raw word count of the model response.
  - `content_quality` (score, 0–1) — heuristic: 0.0 = empty / <5 words, 0.5 = refusal phrase detected, 1.0 = substantive reply.
- Both metrics carry `feature:<featureName>` and `provider:<provider>` tags.
- Wrapped in try/catch — metric failure never blocks the AI response.
- Only fires when `_llmobsEnabled` is true (i.e. `DATADOG_API_KEY` is set).

## inspect-ai-keys Deployment Fix

**Status:** Done ✅ (main agent)

### Root cause (three separate bugs)

1. **Wrong tarball structure** — archive was built with `tar ... inspect-ai-keys/`, wrapping everything under `inspect-ai-keys/`. Appwrite looked for `src/main.js` at root and found nothing. Error: `"file src/main.js does not exist"`. Deployments showed `status: ready` (build succeeded) but all executions returned 503. Fix: `tar -czf -C inspect-ai-keys .` — `src/main.js` now at root.

2. **Missing `APPWRITE_API_KEY` function variable** — function could not instantiate the `Databases` client to read/write `app_settings`. Set via SDK `functions.createVariable()`.

3. **`node_modules` not bundled** — Appwrite's build step did not run `npm install`. Error: `"Cannot find module 'node-appwrite'"`. Fix: `npm install` inside `appwrite-hubs/inspect-ai-keys/`, bundled `node_modules` in tarball.

### Result
- Deployment `69fffedf915e4aaaa90d` — status: ready, active: true.
- Smoke test: HTTP 200, `success: true`, 12 key slots, all four providers, `modelOptions` and `nvidiaModels` served.
- `scripts/deploy_hubs.cjs`: added `inspect-ai-keys` to the `hubs` array.

**⚠️ Packaging pattern for all future hub deploys:**
```bash
# Install dependencies first (Appwrite build step does NOT run npm install reliably)
cd appwrite-hubs/<hub-name> && npm install

# Build tarball WITHOUT top-level directory prefix
cd appwrite-hubs && tar -czf <hub-name>.tar.gz -C <hub-name> --exclude='./.git' .

# Verify structure — src/main.js must appear at root, not under <hub-name>/
tar -tzf <hub-name>.tar.gz | head
```
