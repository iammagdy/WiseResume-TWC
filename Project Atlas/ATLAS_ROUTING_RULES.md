# Project Atlas Master Document Routing Rules & Placement Policy

**Last Verified:** 2026-07-03
**Status:** Canonical File Placement Governance
**Location:** `Project Atlas/ATLAS_ROUTING_RULES.md`

---

## 1. Mandatory Pre-Check for AI Agents & Developers

Before creating, moving, or editing any file in this repository:

1. **Consult This Matrix**: Check the [Routing Rules Matrix](#3-routing-rules-matrix) to determine the exact authorized path for the new file.
2. **Root Hygiene Rule**: Never create random Markdown files directly under the repository root (`/`) or `Project Atlas/` root. Only approved control files are permitted in `Project Atlas/` root.
3. **No Code/Deploy Touch**: Documentation changes must never touch application code (`src/`, `api/`, `server/`, `appwrite-hubs/`), test runner configs, scripts, or workflows unless explicitly tasked.

---

## 2. File Type Governance Definitions

* **Living Specs (`Project Atlas/docs/`, `product/`, `architecture/`, `features/`, `deployment/`)**: Canonical specifications detailing verified architecture, DB schemas, API endpoints, and feature requirements. Must be kept up to date.
* **Handover & Operational State (`WHERE_WE_STOPPED.md`, `CURRENT_STATE.md`)**: Active state tracking current production snapshot, where the project stopped, next tasks, and do-not-reopen constraints.
* **QA Strategy & Checklists (`Project Atlas/qa/`)**: Human-written test plans, Playwright/Vitest suite maps, manual checklists, and QA acceptance criteria.
* **Executable Test Code (`tests/`)**: **Outside Atlas**. Runnable Playwright E2E specs (`*.spec.ts`), Vitest unit tests, and test fixtures located in root `tests/`.
* **Generated Test Output (`reports/`)**: **Outside Atlas**. Machine-generated JSON test results, HTML test reports, traces, and audit dumps automatically generated by test runners in root `reports/`.
* **Agent Skills & Playbooks (`Project Atlas/skills/`)**: Modular execution guides detailing step-by-step procedures for agent tasks (e.g. bootstrap, QA validation, safe Appwrite changes).
* **Agent Prompts & Templates (`Project Atlas/prompts/`)**: Reusable system prompts, prompt engineering rules, and task initiation templates.
* **Audit Reports (`Project Atlas/reports/`)**: Point-in-time categorized audit writeups (`ui-ux/`, `performance/`, `devkit/`, `landing/`).
* **Visual Assets (`Project Atlas/assets/`)**: Brand logos, system architecture diagrams, UI screenshots, and mockups.
* **Decision Records (`Project Atlas/decisions/`, `DECISIONS.md`)**: Architectural Decision Records (ADRs) capturing technical choices and trade-offs.
* **Temporary Scratch Space (`Project Atlas/temp/`)**: Short-lived draft files or scratch scripts used during an active turn. MUST be deleted or promoted to living specs before task completion.
* **Historical Archive (`Project Atlas/archive/`)**: Zero-data-loss repository for stale, superseded, or legacy documentation. **STRICTLY NON-CANONICAL**. Agents MUST NOT use archived files as current truth unless explicitly instructed by the owner.

## 4. Root Directory Hygiene Rules

### Allowed Root Files in `Project Atlas/`
Only the following core control and navigation files are permitted directly under `Project Atlas/`:
* `README.md` — Master navigation directory
* `MASTER_HANDBOOK.md` — AI agent operating manual
* `CURRENT_STATE.md` — Verified production system snapshot
* `WHERE_WE_STOPPED.md` — Active handover and operational state
* `RULES.md` — Developer rules & definition of done
* `ATLAS_ROUTING_RULES.md` — Master routing policy (this file)
* `SOURCE_OF_TRUTH_MAP.md` — Master inventory index map
* `CHANGELOG.md` — System & docs changelog
* `GOVERNANCE.md` — Governance rules
* `MAINTENANCE.md` — Maintenance policy
* `DECISIONS.md` — Current main decision log
* `MASTER_HANDOVER_2026.md` — Chronological session handover history log

### Prohibited Root Actions
* DO NOT create random `.md` files in the repository root (`/`).
* DO NOT create unclassified `.md` audit files or task logs in `Project Atlas/` root.
* DO NOT place scratch scripts or raw log files in `Project Atlas/` root.
